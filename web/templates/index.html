Startup — 

      
Bring the cluster back online.


      

        VMs start in the reverse order of shutdown: the last group stopped is the first to come back up.
        Within each group, all VMs start in parallel and are waited on together before the next group fires.
        Edit groups from the Shutdown tab — the same configuration drives both.
